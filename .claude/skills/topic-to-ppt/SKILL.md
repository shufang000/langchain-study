# 主题生成 PPT 技能

## 工作流

当用户提供一个主题时，按以下步骤执行：

### 第 1 步：主题分析与内容规划

根据用户给定的主题，首先进行内容拆解：

1. **明确受众**：这是给谁看的？（领导汇报、客户演示、团队分享、对外演讲）
2. **确定目标**：这份 PPT 要达到什么目的？（说服、汇报、培训、展示）
3. **拆解大纲**：将主题拆解为 5-10 张幻灯片的结构

**幻灯片结构模板**：

| 页码 | 类型 | 内容要点 |
|------|------|----------|
| 1 | 封面 | 标题 + 副标题 + 演讲者/日期 |
| 2 | 目录/议程 | 本次内容的纲要 |
| 3-N | 内容页 | 每页一个核心观点 |
| 末页 | 总结/致谢 | 关键要点回顾 + 联系方式 |

### 第 2 步：设计风格选择

根据主题属性，自动选择配色和字体方案：

| 主题类型 | 推荐配色 | 风格 |
|----------|----------|------|
| 技术/科技 | 午夜行政 / 海洋渐变 | 深色背景、科技感 |
| 商业/金融 | 炭黑极简 / 信赖青绿 | 专业稳重 |
| 医疗/健康 | 宁静灰绿 / 信赖青绿 | 干净清新 |
| 创意/艺术 | 珊瑚活力 / 温暖陶土 | 大胆活泼 |
| 教育/培训 | 森林苔藓 / 莓果奶油 | 温暖友好 |
| 数据/分析 | 午夜行政 / 炭黑极简 | 理性克制 |

### 第 3 步：调用 pptx skill 生成

确定大纲和风格后，调用 **pptx** skill 进行实际的 PPT 创建。遵循 pptx skill 中的：
- 设计理念（配色、字体、间距、视觉元素）
- 创建方式（从零创建或基于模板）
- QA 流程（内容检查 + 视觉检查）

### 第 4 步：质量检查

生成完成后，按照 pptx skill 的 QA 要求进行检查：
1. 内容 QA：检查遗漏、错别字、顺序
2. 视觉 QA：检查重叠、溢出、对齐、对比度

---

## 注意事项

- 如果用户提供的主题信息不足（没有指定受众、页数等），主动询问关键信息
- 每页幻灯片只聚焦一个核心观点，避免信息过载
- 优先使用图标、图表、数据可视化，减少纯文字页面
- 生成前先向用户展示大纲，确认后再开始制作
